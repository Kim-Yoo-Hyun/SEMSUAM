# Paper Card

## Problem

### 사실

ConceptGraphs는 posed RGB-D image sequence에서 object-centric open-vocabulary 3D scene graph를 만드는 문제를 다룬다. 입력 pose는 주어져 있으며, active SLAM이나 viewpoint selection 자체를 풀지는 않는다.

### 논문 주장

Robotics용 3D representation은 semantically rich, compact, task-driven perception/planning에 효율적이어야 한다. Dense per-point feature map은 large environment에서 memory가 커지고, object/entity 사이 semantic spatial relationship을 직접 담지 못해 downstream planning에 불리하다고 주장한다. 기존 3D scene graph는 compact하지만 closed-vocabulary라 novel object/concept에 약하다고 본다.

### 에이전트 추론

P11은 `CAND-01`의 semantic memory backbone 후보이다. 직접적인 active re-observation 논문은 아니지만, object node, caption, CLIP feature, spatial edge를 active uncertainty / ObjectNav evaluation에 연결할 수 있다.

## Core Idea

2D foundation models로 RGB image에서 object candidates와 semantic features를 만들고, depth/pose로 3D point cloud에 투영한 뒤 multi-view geometric + semantic association으로 object nodes를 fuse한다. 이후 LVLM/LLM으로 node caption과 inter-object relation을 생성해 open-vocabulary 3D scene graph를 만들고, JSON/text interface로 LLM-based task planning에 연결한다.

## Input / Output

Input:

- posed RGB-D sequence
- class-agnostic or open-vocabulary 2D segmentation/detection outputs
- CLIP / DINO-style visual descriptors
- depth-projected object point clouds
- LVLM captions and LLM relation/planning prompts

Output:

- object-centric 3D map
- 3D scene graph with object nodes and relationship edges
- object captions, object tags, CLIP features, 3D bounding boxes / centers
- JSON/text scene description for LLM planner
- downstream outputs for object retrieval, navigation target, manipulation target, localization/remapping, and traversability decision

## Method

### 사실

- Each object node stores a 3D point cloud and semantic feature vector.
- Each RGB-D frame is segmented with a class-agnostic segmentation model.
- The paper experiments use `SAM` for segmentation and `CLIP` image encoder for semantic feature extraction.
- Each mask is projected into 3D using depth and pose, denoised with `DBSCAN`, and transformed into the map frame.
- New detections are associated with existing objects using a sum of geometric similarity and semantic similarity.
- Geometric similarity is based on nearest-neighbor overlap within a threshold.
- Semantic similarity uses normalized cosine similarity between visual descriptors.
- Greedy object association is used; the paper says Hungarian matching was slower with only small improvement.
- Object fusion updates semantic features and point clouds, then downsamples to remove redundant points.
- Node captioning uses up to the 10 most informative object views.
- The paper uses `LLaVA` for per-view object captions and `GPT-4` (`gpt-4-0613`) to summarize captions into object tags/captions.
- Scene graph edges are proposed from 3D bounding box IoU and pruned with a minimum spanning tree.
- Semantic relationship labels are generated by prompting an LLM with object captions and 3D locations.
- For planning, each object is converted to JSON containing object id, bounding box extent, bounding box center, object tag, and caption.
- `ConceptGraphs-Detector` / `CG-D` variant uses `RAM`, `Grounding DINO`, and `SAM`.

### 논문 주장

- Object-centric graph structure is more compact and planner-friendly than dense point-level feature maps.
- Off-the-shelf 2D foundation models can provide open-vocabulary semantics without 3D finetuning.
- LLM/LVLM captions and relations enable abstract language queries and complex reasoning over objects.
- The same representation supports perception, planning, navigation, manipulation, localization, and map updates.

### 에이전트 추론

For `CAND-01`, the immediate reusable unit is not the whole LLM planner but the semantic memory schema:

- node: object point cloud / bbox / caption / CLIP feature / view history
- edge: spatial or affordance relation
- uncertainty candidate: low caption agreement, low multi-view support, conflicting features, duplicate candidate, missing small/thin object risk

This schema can host Step 1 object/node uncertainty and Step 2 active re-observation viewpoint selection.

## Main Claims

### 논문 주장

- ConceptGraphs discovers, maps, and captions many objects in a scene.
- Node labels are accurate about 70% of the time and spatial relationship edges about 90% on average in the reported AMT evaluation.
- ConceptGraphs achieves open-vocabulary semantic segmentation performance comparable to or better than `ConceptFusion`, with a smaller memory footprint.
- LLM-based retrieval over scene graph captions handles affordance and negation queries better than CLIP-only retrieval.
- Real-world Jackal and Spot demonstrations show broad planning and manipulation utility.

## Strengths

- Strong fit to environmental perception intelligence as structured semantic memory.
- Official project page and code are available.
- Evaluation includes graph quality, semantic segmentation, text-query retrieval, and real-world robot demonstrations.
- Directly exposes node captions, object tags, bounding boxes, and spatial relations that can be reused in `CAND-01`.
- The codebase now includes a real-time refactored branch and AI2Thor-related support, which may help practical harness building.
- Real-world demonstrations with Jackal and Spot are relevant to later deploy planning.

## Limitations

### 사실

- The method assumes posed RGB-D input.
- It does not optimize pose or handle active SLAM uncertainty.
- It does not evaluate ObjectNav `Success Rate`, `SPL`, wrong-goal visit, or wasted path.
- It does not report `ATE`, `RPE`, map error, or pose graph connectivity.
- The paper states node captioning can fail due to `LLaVA` limitations.
- The paper states the graph can miss small/thin objects and make duplicate detections.
- The paper states LVLM and proprietary LLM inference costs can be significant.
- The paper does not explicitly focus on improving LLM planning over scene graphs.

### 에이전트 추론

- Node/edge errors are exactly the kind of semantic uncertainty that `CAND-01` can exploit, but ConceptGraphs itself does not define an active correction policy.
- Since pose is assumed, direct use as Step 4-5 SLAM uncertainty baseline is weak.
- Full reproduction has dependency cost: `SAM`, `Grounded-SAM`, `LLaVA`, `GPT-4`, `Pytorch3D`, `GradSLAM`, dataset conversion, and possible API cost.

## Relevance to My Research

### 사실

`CAND-01` aims to use pre-explored semantic memory, select active re-observation viewpoints, evaluate ObjectNav behavior, and later extend semantic memory into active SLAM utility.

### 에이전트 추론

P11 is a good semantic memory substrate for Step 1-3. It provides object nodes and relations that can be assigned uncertainty and revisited. It is less useful as a direct Step 4-5 localization uncertainty paper, but it can supply the semantic graph side of a semantic-SLAM joint utility.
